Git从入门到放弃

发表于 2017-02-26

Git

忽略文件

1.添加忽略文件
在使用Git时,我们通常会使用到.gitignore文件,用来忽略一些我们不想要提交的文件.我们可以根据不同的需求制定不同的.gitignore文件;
以笔者Android开发的忽略文件为例:
忽略文件在AndroidStudio如下:

/build
*.iml
/local.properties
app/build/
/.idea
/build
/captures
gradle/

在添加完成后,将忽略文件也一同提交远程仓库就可以了;
2.修改忽略文件
在我们项目中添加了.gitignore后,但是却发现有些忽略规则忘记添加了,那我们就需要修改我们的忽略文件(.gitignore)了;
在原来的忽略文件(.gitignore)后,在当前仓库的根目录下,执行以下git命令:

git rm -r --cached . //清除缓存
git add .``//重新添加文件索引
git commit -m "更新.gitignore文件" //写注释并提交
git push origin master //同步到远程分支是mater的仓库

这样,我们的忽略文件(.gitignore)就更新成功了;
