Realm数据库入门

Posted on 2017-05-23 | In Android , Database | Visitors

前言

最近项目需要用到数据库,想到以前使用的GreenDao集成比较麻烦.不过Greendao从3.0以后支持注解操作,所以集成起来还是比较方便的.
最近在网上看到有一个新的数据库框架,该框架不同于以前版本的数据库框架,她不是基于SQLite而封装了.而是底层采用了C++新写的一项嵌入式数据库,支持运行在手机、平板和可穿戴设备上.她不用于SQLite,她允许你在持久层直接和数据对象工作。在它之上是一个函数式风格的查询api，众多的努力让它比传统的SQLite 操作更快 。
说了半天, 她就是Realm.由Y Combinator公司开发出来;
敲黑板:目前Realm支持Java,OC,RN,Swift,JS等语言,是不是想起来都很牛叉.那下面让我们来一起学习吧;

  • @primarykey(表示该字段是主键)
    PrimaryKey就是主键。使用@PrimaryKey来标注，字段类型必须是字符串（String）或整数（byte，short，int或long）以及它们的包装类型不可以存在多个主键，使用字符串字段作为主键意味着字段被索引（注释@PrimaryKey隐式地设置注释@Index）。

  • @Required(表示该字段非空)
    在某些情况下，有一些属性是不能为null的。使用@Required可用于用于强行要求其属性不能为空，只能用于Boolean, Byte, Short, Integer, Long, Float, Double, String, byte[] 和 Date。在其它类型属性上使用 @Required修饰会导致编译失败。
    Tip：基本数据类型不需要使用注解 @Required，因为他们本身就不可为空。

  • @Ignore(表示忽略该字段)
    被添加@Ignore标签后，存储数据时会忽略该字段。

坑

  1. 主键重复异常
    Primary key constraint broken. Value already exists: 0
    这是因为 Realm.createObject(Class clazz) 实际上隐式调用了原始对象的默认无参数构造器，然后通过 Realm.copyToRealm() 方法将其存入 Realm 中。隐式构造器会给其主键属性赋一个默认值，而当第二次调用时，主键仍会是这个默认值。这就导致了 Realm 存储的对象出现了重复主键，从而异常被抛出。解决方法有很多种，譬如调用Realm.createObject(Class clazz, Object primaryKeyValue) 方法在对象创建时指定一个不重复的主键。
  2. 字段修饰符已存在
    Invalid type 'Long' for field 'id' in existing Realm file.
    这个是我们在编码的过程中,由原本的String id 改成了 Long id,那么就会报出这样的BUG,这里我的解决方法,就是删除原本手机中的数据库,即可.
