采用Hexo和Gitpage搭建自己的博客

Posted on 2017-02-24 | In Tools , Hexo | Visitors

前言

如何搭建

搭建的教程建议看参考文档,写的非常详细;
注意

初始化博客

在安装好hexo后,执行以下命令
hexo init blog
tips:在 hexo 目录下生成blog文件夹，包含 node_modules、scaffolds、source、themes、.gitignore、_config.yml、db.json、package.json，其中 _config.yml 为全局配置文件

##启动服务
hexo server
出现提示:
INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.
此时在浏览器输入: http://localhost:4000/ 既可以访问本地hexo数据;
如果不能看到,检查地址,查看hexo是否安装正确,直到正确为止;

##常用命令

######1.新建文件
hexo new "文件名" – 在/source/_posts/创建 文件名.md 文件;

######2.新建文件夹：
hexo new page "文件名" – 在/souce 创建文件夹,并在内部会默认生产index.md 文件;

######3.新建文件和依赖的文件夹:
hexo new host "文件名" – 在/source/_posts 创建 “文件名”.md 和 一个相同名字的文件夹
tips:在这个文件夹中可以存放相应的图片和资源文件,在相同文件名md文件中通过相对应的url可以连接到这个文件夹中的资源文件;

######4.提交到git:
分别执行以下命令;

hexo clean//清除缓存
hexo generate//生成public文件,准备提交
hexo deploy//提交
–以上命令执行之后,稍等,就可以在github上查看提交的信息了;
